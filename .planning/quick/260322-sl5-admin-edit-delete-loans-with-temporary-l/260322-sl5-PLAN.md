**Temporary privilege concept:** The loan officer sees edit/delete buttons ONLY on the loan they just created, and ONLY until they navigate away or refresh. This is implemented via client-side state:
- `createLoanAction` in the new-loan page already redirects to the loan detail after creation
- On the loans LIST page, loan officers do NOT see edit/delete (only admin+ do)
- On the loan DETAIL page, a `canModify` prop is passed from the server component. For admin+, it's always true. For loan officers, the client component receives a `freshlyCreated` prop that is `true` only when the page was navigated to from the loan creation flow (use a URL search param `?new=1` set by the create loan redirect). Once the user navigates away or reloads (no `?new=1`), the buttons disappear.

export async function getCurrentUserRoleAction(): Promise<UserRole> {
  const session = await auth.api.getSession({ headers: await headers() })

}
