## DrawerDialog Visual Specification

### Desktop (md+, 768px and above)

- Renders as centered modal
- Position: `fixed top-1/2 left-1/2 -translate-x-1/2 -translate-y-1/2`
- Max width: `sm:max-w-sm` (match existing DialogContent)
- Background: `bg-white/85`
- Border radius: `rounded-lg` (0.5rem from --radius)
- Padding: `p-4` (16px)
- Overlay: `bg-black/10 backdrop-blur-[24px]`
- Close button: `Button size="icon-sm"` at `absolute top-2 right-2`
- Animation: `data-open:animate-in data-open:fade-in-0 data-open:zoom-in-95` / `data-closed:animate-out data-closed:fade-out-0 data-closed:zoom-out-95`

### Mobile (< md, below 768px)

- Renders as bottom drawer
- Position: `fixed bottom-0 inset-x-0`
- Max height: `max-h-[90dvh] overflow-y-auto`
- Background: `bg-white/85`
- Border radius: `rounded-t-xl` (top corners only)
- Padding: `p-4` (16px)
- z-index: `z-50` (above BottomTabBar at z-40)
- data-slot: `drawer-dialog-content`
- Drag handle: `<div className="mx-auto mb-4 h-2 w-12 rounded-full bg-muted-foreground/30" />` at top of popup
- Overlay: `bg-black/10` fixed inset-0
- swipeDirection: `"down"` on Drawer.Root
- Animation: `data-open:animate-in data-open:slide-in-from-bottom` / `data-closed:animate-out data-closed:slide-out-to-bottom`
- Swipe-to-close: enabled via Drawer.Root swipeDirection="down" (no explicit SwipeArea needed for DrawerDialog — swipe on the popup itself closes it)

### SSR Safety

- Use `useMediaQuery("(min-width: 768px)", { defaultMatches: true, noSsr: false })` from `@base-ui/react/unstable-use-media-query`
- `defaultMatches: true` means server render defaults to Desktop (Dialog) path — avoids bottom-drawer flash on desktop initial load
- `noSsr: false` allows re-render after hydration to sync with actual viewport on mobile

## Touch Target Specification (TOUCH-01)

All interactive elements at mobile viewport (< 768px) must have a minimum 44x44px tappable area per WCAG 2.5.8. The visual size may remain smaller — use `min-h`/`min-w` to expand the hit area without changing desktop appearance.

### Button Component Updates

| Size variant | Current visual | Mobile hit area | Tailwind classes to add |
|---|---|---|---|
| default | h-8 (32px) | 44px height | `min-h-[44px] md:min-h-0` |
| lg | h-9 (36px) | 44px height | `min-h-[44px] md:min-h-0` |
| icon | size-8 (32x32px) | 44x44px | `min-h-[44px] min-w-[44px] md:min-h-8 md:min-w-8` |
| icon-sm | size-7 (28x28px) | 44x44px | `min-h-[44px] min-w-[44px] md:min-h-7 md:min-w-7` |
| icon-lg | size-9 (36x36px) | 44x44px height | `min-h-[44px] min-w-[44px] md:min-h-9 md:min-w-9` |
| sm | h-7 (28px) | Apply only if appears in mobile card actions | Audit at task time |
| xs | h-6 (24px) | Apply only if appears in mobile card actions | Audit at task time |

**Established project pattern** (notification bell in `notification-bell.tsx:99`):
```
className="relative min-h-[44px] min-w-[44px]"
```
Use this exact pattern for all fixes.

## MoreSheet Swipe-to-Close Specification (TOUCH-03)

**Scope:** Swipe-down-to-close on the MoreSheet drawer only. Horizontal page swipe navigation is explicitly out of scope (conflicts with browser back gesture, BottomTabBar already handles navigation).

**Migration:** Convert `more-sheet.tsx` from `Sheet` (Dialog-based) to `@base-ui/react/drawer`:

| Before | After |
|--------|-------|
| `Sheet` wrapping `@base-ui/react/dialog` | `Drawer.Root` from `@base-ui/react/drawer` |
| `SheetContent side="bottom"` | `Drawer.Popup` with `swipeDirection="down"` on root |
| No drag handle | `Drawer.SwipeArea` + visual drag handle div at top |
| No native swipe gesture | Swipe-down dismisses drawer |

**Visual drag handle** inside `Drawer.Popup` at top:
```
<div className="mx-auto mb-2 h-2 w-12 rounded-full bg-muted-foreground/30" />
```

**z-index:** Drawer backdrop `z-50`, Drawer popup `z-50` — matches existing Sheet and Dialog; correctly above BottomTabBar at `z-40`.

## Interaction States

### DrawerDialog

| State | Desktop behavior | Mobile behavior |
|-------|-----------------|-----------------|
| Closed | Dialog unmounted | Drawer unmounted |
| Opening | zoom-in-95 + fade-in-0 animation (~100ms) | slide-in-from-bottom animation (~200ms) |
| Open | Centered modal, focus trapped | Bottom drawer, focus trapped, drag handle visible |
| Closing by button | zoom-out-95 + fade-out-0 (~100ms) | slide-out-to-bottom (~200ms) |
| Closing by swipe | N/A (desktop) | Velocity-based swipe-down dismisses; partial drag snaps back |
| Closing by backdrop click | Dialog closes | Drawer closes |
| Closing by Escape key | Dialog closes | Drawer closes |

### Touch Targets

| State | Behavior |
|-------|----------|
| Default | Visual size unchanged; hit area expanded to 44x44px on mobile via min-h/min-w |
| Hover | bg-muted on DropdownMenuTrigger (existing) |
| Active | `active:translate-y-px` on Button (existing, via buttonVariants base class) |
| Focus-visible | `ring-3 ring-ring/50` Electric Blue ring (existing, via buttonVariants base class) |
| Disabled | `opacity-50 pointer-events-none` (existing) |
