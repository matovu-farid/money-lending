**1. Create `src/app/(app)/payments/LoanSearchCombobox.tsx`**

"use client" component. Props:
```typescript
interface LoanSearchComboboxProps {
  selectedLoan: ActiveLoanSearchResult | null
  onSelect: (loan: ActiveLoanSearchResult) => void
  onClear: () => void
}
```

Implementation:
- State: `query` (string), `open` (boolean), `results` (ActiveLoanSearchResult[]), `isSearching` (boolean)
- On input change: set query, if query.trim().length >= 2 start 200ms debounce timer (clear previous timer with `useRef<ReturnType<typeof setTimeout>>`), then call `searchActiveLoansAction(query)`, set results from response data
- Open popover when query.length >= 2 and results exist or isSearching
- When a result is clicked: call `onSelect(result)`, close popover, clear query
- When selectedLoan is non-null: show selected loan name in input as read-only text with an X clear button (lucide-react X icon) with `aria-label="Clear loan selection"`
- Search result row format: `{customerName}  ·  LOAN-{loanId.slice(0,8).toUpperCase()}` on first line, `Balance: UGX {formatNumberWithCommas(principalAmount)}` on second line (Caption style: text-xs text-muted-foreground)
- Popover uses `<PopoverTrigger render={<div className="relative w-full" />}>` pattern (base-ui render prop, NOT asChild)
- PopoverContent: `sideOffset={4}`, `className="w-[var(--anchor-width)] p-0"` to match input width
- Loading state: show Spinner centered in PopoverContent with text "Searching..."
- Empty state: "No active loans found for \"{query}\""
- Below 2 chars state: "Type at least 2 characters to search"
- Search icon (lucide-react Search) positioned absolute right inside the input

**2. Create `src/app/(app)/payments/QuickRecordDialog.tsx`**

"use client" component. Props:
```typescript
interface QuickRecordDialogProps {
  open: boolean
  onOpenChange: (open: boolean) => void
}
```

Implementation:
- State: `selectedLoan` (ActiveLoanSearchResult | null), `amount` (string), `paymentDate` (string, default `format(new Date(), "yyyy-MM-dd")` from date-fns), `successPaymentId` (string | null), `isPending` via useTransition
- Recently-collected chips: `useQuery({ queryKey: ["recentLoans"], queryFn: async () => { const r = await getRecentlyCollectedLoansAction(); return "error" in r ? [] : r.data }, staleTime: 0 })`
- Recently-collected section: only render if recentLoans array is non-empty. Label "Recent" (text-sm font-semibold). Chips rendered as `<Button variant="secondary" size="sm">` in a `flex flex-wrap gap-2` container. Each chip: `{customerName} · LOAN-{loanId.slice(0,8).toUpperCase()}`
- Clicking a chip: construct an ActiveLoanSearchResult from the chip data and call setSelectedLoan
- LoanSearchCombobox rendered below chips with Label "Loan"
- Amount field: `type="number"` input, `min="1"`, `step="1"`. Label "Amount (UGX)". UGX prefix via relative/absolute positioning pattern: `<div className="relative"><span className="absolute left-3 top-1/2 -translate-y-1/2 text-sm text-muted-foreground">UGX</span><Input className="pl-12" ... /></div>`. Disabled when no loan selected.
- Payment date: `type="date"` input, Label "Payment Date", default today
- Submit button: "Record Payment" (default variant), disabled when `!selectedLoan || !amount || Number(amount) <= 0 || isPending`. Shows Spinner + "Recording..." text when isPending.
- On submit: `startTransition(async () => { const result = await recordPaymentAction({ loanId: selectedLoan.loanId, amount, paymentDate: new Date(paymentDate).toISOString() }); if ("error" in result) { toast.error(result.error === "Loan not found" ? "Loan not found. It may have been deleted." : "Failed to record payment. Please try again."); return; } setSuccessPaymentId(result.data.id); queryClient.invalidateQueries({ queryKey: ["payments"] }); queryClient.invalidateQueries({ queryKey: ["recentLoans"] }); })`
- Success state (when successPaymentId is non-null): replace form content with:
  - DialogTitle changes to "Payment Recorded"
  - Body text: "Payment of UGX {formatNumberWithCommas(amount)} recorded for {selectedLoan.customerName}."
  - Link: `<a href="/receipts/repayment/${successPaymentId}" target="_blank" className="text-sm underline-offset-4 hover:underline">View receipt</a>` (use Next.js Link if preferred)
  - Footer: "Record another" (default variant) and "Close" (outline variant)
  - "Record another" handler: `setSelectedLoan(null); setAmount(""); setPaymentDate(format(new Date(), "yyyy-MM-dd")); setSuccessPaymentId(null);`
- Close/onOpenChange handler: when dialog closes, reset all state (same as "Record another" reset)
- DialogFooter in idle state: "Close" (outline) on left, "Record Payment" (default) on right
