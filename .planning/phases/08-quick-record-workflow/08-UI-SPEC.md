## Interaction Rules

### LoanSearchCombobox
- Open popover when input has >= 2 characters (not on focus alone)
- Debounce search call: 200ms after last keystroke
- Show `Spinner` inside popover while results loading
- Show max 10 results
- On result click: close popover, set selected loan, display selected loan name in input field (read-only appearance), enable amount field
- Clear button (X icon) visible when a loan is selected — clicking resets selection and re-enables typing
- PopoverContent renders with `sideOffset={4}`; set `style={{ zIndex: 9999 }}` if z-index conflict with Dialog overlay is detected during implementation

### Recently-Collected Chips
- Rendered as `<Button variant="secondary" size="sm">` elements (not Badge)
- Displayed as a horizontal wrapping flex row with `gap-2`
- Maximum 5 chips shown; oldest drops off after successful payment
- Clicking a chip: sets selectedLoan state, pre-fills loan field exactly as search selection would
- Section hidden entirely (no empty state label) when `recentLoans` array is empty

### Form Fields
- Amount field: `type="number"` input with `min="1"` and `step="1"` — no decimal payments; prepend "UGX" as a non-interactive prefix label to the left of the input using `relative` + `absolute` positioning pattern (same as existing filter bar)
- Payment date: `type="date"` input defaulting to today's date (`format(new Date(), "yyyy-MM-dd")`)
- Submit button: disabled when no loan selected, amount is empty or <= 0, or `isPending` is true
- Submit shows `Spinner` icon inside button while `isPending` (using `useTransition`)

### Success State Transition
- Do NOT auto-close dialog after success
- Switch to success state view inline — same dialog, replace form content
- "Record another" resets all state: selectedLoan → null, amount → "", paymentDate → todayStr, query → "", successPaymentId → null
- Receipt link opens in new tab (`target="_blank"`)
- TanStack Query invalidation on success: `queryKey: ["payments"]` and `queryKey: ["recentLoans", userId]`

### Trigger Button Placement
- "Record Payment" button placed in PaymentsClient header bar, to the right of the existing filter controls
- Accessible from both "List" and "Daily" tabs (button is in the shared header above the tabs)
- Button: `variant="default"` (primary), `size="default"`, label: "Record Payment"

## States & Transitions

| State | Trigger | Visual Change |
|-------|---------|---------------|
| Dialog closed | Initial | Button visible in header |
| Dialog open — no loan selected | Click "Record Payment" | Dialog renders, amount + submit disabled |
| Dialog open — chips visible | recentLoans fetched, array non-empty | Chips row renders above search field |
| Loan selected via chip | Chip click | Input shows loan name; X clear button appears; amount enabled |
| Loan selected via search | Result row click | Popover closes; input shows loan name; X clear button appears; amount enabled |
| Searching | 2+ chars typed, debounce elapsed | Popover opens with Spinner |
| Results loaded | Action returns | Spinner replaced by result rows |
| No results | Action returns empty array | "No active loans found" message |
| Submit pending | Submit click | Button disabled + Spinner; form fields disabled |
| Success | Action returns { data } | Form replaced by success view |
| Dialog closed from success | Close or backdrop | Dialog unmounts; form state reset |
| Record another | "Record another" click | Success view replaced by fresh form |
