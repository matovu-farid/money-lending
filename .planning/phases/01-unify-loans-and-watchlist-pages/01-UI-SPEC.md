## Interaction Contract

### Criticality Sort (default, client-side)

```
criticalityRank: daysOverdue >= 30 в†’ 0, >= 15 в†’ 1, >= 1 в†’ 2, else в†’ 3
Sort: rank asc, then daysOverdue desc within group
```

Applied before filtering. Filter narrows the already-sorted list.

### Filter Behavior

- `"all"`: show all loans (all statuses, all overdue levels), count = total loan count
- `"critical"`: show loans with daysOverdue >= 30
- `"at-risk"`: show loans with daysOverdue >= 15 and < 30
- `"early"`: show loans with daysOverdue >= 1 and < 15
- Current loans (daysOverdue === 0) appear in "all" only вЂ” no dedicated filter tab

Clicking a stat card activates that card's matching filter tab. State is colocated in the page component via `useState<FilterCategory>`.

### Print Behavior

- `Button "Print"` calls `window.print()`
- Elements hidden during print: sidebar, header action row (New Loan, Print buttons), stat cards, filter tabs
- Apply `print:hidden` Tailwind variant on those elements
- Table renders with current filter applied (not full list) вЂ” print the view the user sees
- Global print CSS in globals.css already sets `--background: white`, `--foreground: black`

### Row Navigation

- Table row `onClick`: `router.push(\`/loans/${entry.id}\`)`
- Customer Name link `href`: `/customers/${entry.customerId}` with `ev.stopPropagation()`
- "New Loan" button: `href="/loans/new"` (Link or router.push)
- No modal or sheet opens from this page вЂ” all interactions are navigational

### Loading State

3 skeleton rows: `border rounded-md p-4 space-y-2` with two pulse bars per row (`h-4 w-32 animate-pulse bg-muted-foreground/10 rounded`). Matches existing watchlist loading pattern.
