## Screen-Level Interaction Contracts

### 1. Executive Dashboard (`/dashboard`)

**Layout:** Two rows of 3 KPI cards (6 total), then activity feed below.

**KPI cards:**
- Row 1: Loans Outstanding (UGX), Repayments Collected (UGX), Interest Earned (UGX)
- Row 2: Active Borrowers (count), Overdue Count (count), Capital in System (UGX â€” shows "UGX 0" with muted subtitle "Creditor data available in Phase 4")
- Each card: icon top-right (lucide), label at 14px regular, value at 28px semibold, no interactive state (display only)
- Grid: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-6`
- Overdue Count card: if count > 0, value text uses `text-destructive`; if 0, uses default foreground

**Activity feed:**
- Title: "Recent Activity" at 20px semibold
- Shows last 10 events (payments received, loans issued, overdue flags)
- Each item: event icon (lucide) left, event description and timestamp right, separated by `border-b`
- Timestamp: relative ("2 hours ago") in `text-muted-foreground` at 14px
- Empty state: "No recent activity yet." (see Copywriting Contract)

### 2. Customer Search & Filtering (`/customers`)

**Toolbar above table:**
- Search input (placeholder: "Search by nameâ€¦") with lucide `Search` icon left of input â€” width: `max-w-sm`
- Status filter: Select component with options: All Statuses / Active / Blacklisted / Inactive
- Loan status filter: Select component with options: All / Pending / Active / Fully Paid
- Days remaining filter: Select component with options: Any / Due within 30 days / Overdue (30+ days)
- Toolbar layout: `flex flex-wrap gap-3 items-center` â€” search bar first, dropdowns right-aligned on desktop

**Pagination:**
- Show "Showing Xâ€“Y of Z customers" in `text-sm text-muted-foreground`
- Previous / Next buttons; disabled state when at first/last page
- Page size: 20 per page (server-side)

**Empty state** (no results matching filter): "No customers match your search." with "Clear filters" link button.

**Status badge on rows:** uses existing `statusVariant()` pattern from `customers/page.tsx` â€” `default` (active), `destructive` (blacklisted), `secondary` (inactive).

### 3. Customer Status Change (inline on `/customers/[id]`)

**Pattern:** Status badge on customer profile header becomes a Select dropdown on click.

**Interaction flow:**
1. User clicks the current status badge
2. Dropdown opens with 3 options: Active / Blacklisted / Inactive â€” current status shown with checkmark
3. On selection of a different status, a confirmation Dialog opens
4. Dialog contains: "Change status to [New Status]?", a Textarea for reason (required, min 10 chars), "Confirm" (primary) and "Cancel" buttons
5. On confirm: Server Action fires, toast success, status badge updates
6. Blacklist selection: Dialog title reads "Blacklist [Customer Name]?" â€” uses `text-destructive` on dialog title

**Blacklist safeguard:** If user attempts loan issuance for a blacklisted customer, loan wizard step 1 shows inline error: "This customer is blacklisted and cannot receive new loans."

### 4. Customer Loan History (on `/customers/[id]`)

**Layout:** Below the customer details section, a "Loan History" section heading at 20px semibold.

**Loan cards:** Each loan renders as a Card component:
- Card header: Loan reference, status badge, issue date
- Card body: Principal amount (28px semibold), outstanding balance
- Expand/collapse toggle: chevron icon right-aligned â€” rotates 180Â° when expanded
- Expanded state: Table of payments with columns: Date, Amount, Interest Portion, Principal Portion, Balance After
- Deleted payments shown with `line-through text-muted-foreground` (established Phase 2 pattern)

**Empty state** (no loans): "No loans on record for this customer." (see Copywriting Contract)

### 5. Borrower Watchlist (`/watchlist`)

**Table columns:** Customer Name | Loan Amount | Outstanding Balance | Days Overdue | Daily Rate (UGX) | Last Payment

**Days Overdue:** `OverdueBadge` component â€” shows numeric value + visual color coding per badge contract above.

**Sort order:** Default sort by days overdue descending (most overdue first).

**Real-time indicator:** Page subtitle reads "Last calculated: [timestamp of page load]" in `text-muted-foreground` at 14px.

**Loading state:** Skeleton rows (3 rows) while server calculates overdue values.

**Empty state** (no flagged borrowers): "No borrowers on the watchlist. All loans are within the 30-day threshold." (see Copywriting Contract)

**Row interaction:** Clicking a row navigates to the customer profile page (`/customers/[id]`).

### 6. Repayment Simulator (inline on `/loans/[loanId]`)

**Placement:** Below the payments table on the loan detail page. Section heading: "Repayment Simulator" at 20px semibold.

**Input:** Single Input field labeled "Simulate payment of UGX" â€” numeric input, no decimals, placeholder: "0".

**"Simulate" button:** Triggers client-side calculation (no network call). Primary button variant.

**Before/after display:** Two Cards side by side (`grid grid-cols-2 gap-6`) labeled "Current" and "After Simulated Payment":
- Each card shows: Outstanding Balance, Unpaid Interest, Days Coverage
- "After" card: changed values highlighted with `font-semibold` where different from current

**Edge cases:**
- Amount = 0: "Enter an amount to simulate." inline message below input in `text-muted-foreground`
- Amount exceeds outstanding balance: "This amount would fully pay off the loan." shown in `text-green-700` below the after-card
- Amount < interest owed: "This amount covers only partial interest. No principal is reduced." shown in `text-yellow-700`

**Days coverage label:** "Days coverage after payment: X days" â€” if result is 0 or negative: "Loan is current â€” no overdue days."

### 7. Notification Bell (top bar)

**Placement:** Right side of top bar, `NotificationBell` client island replacing the current empty right side of `TopBar`.

**Bell states:**
- No unread: `Bell` lucide icon, `text-muted-foreground`
- Has unread: `BellDot` lucide icon variant OR `Bell` with `Badge` overlay showing count â€” `bg-primary text-primary-foreground`, positioned absolute top-right of bell button
- Max display: Shows "9+" if unread count exceeds 9

**Dropdown (Popover):**
- Opens below bell icon, width: `w-80`
- Header row: "Notifications" (label) + "Mark all as read" link button (right-aligned, only shown when unread > 0)
- Each notification item: loan reference, customer name, days until due, relative timestamp
- Item layout: `Loan #[ref] for [Customer] â€” due in X days` at 14px, timestamp below at 12px `text-muted-foreground`
- Unread items: left border `border-l-2 border-primary pl-3`
- Read items: `text-muted-foreground`
- Click on item: marks as read, navigates to loan detail page
- Empty state: "No alerts at this time." in `text-muted-foreground` centered in dropdown
