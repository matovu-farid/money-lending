## Spacing Scale

Declared values (all multiples of 4, sourced from Tailwind v4 default scale):

| Token | px Value | Tailwind Class | Usage |
|-------|----------|----------------|-------|
| xs | 4px | `p-1`, `gap-1` | Icon gaps, inline tight padding |
| sm | 8px | `p-2`, `gap-2` | Compact element spacing, card field row gaps |
| md | 16px | `p-4`, `gap-4` | Card internal padding, default element spacing |
| lg | 24px | `p-6`, `gap-6` | Section padding (desktop only via `md:p-6`) |
| xl | 32px | `p-8`, `gap-8` | Layout column gaps |
| 2xl | 48px | `gap-12` | Major section breaks |
| 3xl | 64px | `gap-16` | Page-level spacing |

**Exceptions (this phase):**

- **Mobile card internal padding:** `p-4` fixed (not `p-4 md:p-6`) — cards are mobile-only elements, no responsive override needed.
- **Card detail grid gaps:** `gap-x-3 gap-y-1` (12px horizontal, 4px vertical) — matches the existing `dl` grid pattern in dashboard activity feed (from RESEARCH.md Pattern 1 code example).
- **Card list vertical spacing:** `space-y-3` (12px between cards) — a deliberate 12px choice to keep the card list dense without using the full 16px md gap.
- **Page-level responsive padding:** `p-4 md:p-6` — established in Phase 11 (RESP-06), preserved unchanged.

## Typography

All sizes use Tailwind's default rem scale at base 16px. Mono font is mandatory for all currency and numeric values.

| Role | Tailwind Class | Size | Weight | Font | Line Height | Usage |
|------|---------------|------|--------|------|-------------|-------|
| Body | `text-sm` | 14px | 400 (regular) | Geist Sans | 1.5 (Tailwind default) | Table cells, card detail values, filter labels |
| Label | `text-xs font-semibold uppercase tracking-wider` | 12px | 600 (semibold) | Geist Sans | 1.5 | Page subtitles, column headers, card `dt` labels |
| Heading | `text-2xl font-semibold tracking-tight` | 24px | 600 (semibold) | Geist Sans | 1.2 | Page `h1` titles (e.g. "Customers", "Loans") |
| Display | `text-xl font-semibold` | 20px | 600 (semibold) | Geist Sans | 1.2 | Section headings, KPI card titles |

**Numeric/currency values:** always `font-mono tabular-nums` (Geist Mono) — applies to amount columns in all 7 list pages and KPI card values. Right-align with `text-right`.

**Heading letter-spacing:** `tracking-tight` (`-0.025em`) — defined in `globals.css` with `-0.02em` on `h1/h2/h3`. Applied via `tracking-tight` on `h1` class.

**Card primary field:** `text-sm font-medium` (14px, weight 500) — the primary (boldest) field within a mobile card. Distinguishable from detail fields but not heading-weight.

## Color

All values are CSS custom properties defined in `src/app/globals.css`. Use Tailwind semantic classes — never hardcode hex.

| Role | CSS Variable | Approximate Hex (light mode) | Usage |
|------|-------------|-------------------------------|-------|
| Dominant (60%) | `--background` = `oklch(0.98 0 0)` | #f9f9fb | Page backgrounds, outer layout surfaces |
| Secondary (30%) | `--card` = `oklch(1 0 0)` | #ffffff | Mobile card shells (`bg-card`), table container, sidebar |
| Muted surface | `--muted` = `oklch(0.96 0 0)` | #f3f3f5 | Hover state on interactive rows (`hover:bg-muted/50`) |
| Accent (10%) | `--ring` = `oklch(0.35 0.2 264)` | #002f9c (Electric Blue) | Focus rings on interactive cards/rows, keyboard navigation |
| Destructive | `--destructive` = `oklch(0.577 0.245 27.325)` | ~#d0341a | Error text in list pages only — not used in Phase 13 interactions |
| Border | `--border` = `oklch(0.8 0 0)` | #c6c6c6 | Card border (`border` class), table row dividers |
| Muted foreground | `--muted-foreground` = `oklch(0.42 0 0)` | #474747 | Card `dt` labels, secondary metadata |

**Accent reserved for:** focus rings only (keyboard-accessible interactive elements). Not used for highlights, tags, or badges in this phase.

**Hover state for interactive cards:** `hover:bg-muted/50 transition-colors` — applies to cards where the entire card is clickable (Customers, Loans, Watchlist). Source: RESEARCH.md Pattern 1 code example.

**Badge colors:** status badges use existing `badge` variants — `default` (primary/black) for active, `destructive` for blacklisted/overdue, `secondary` (muted) for inactive/pending. No new badge variants introduced this phase.

**Dark mode:** all tokens have dark-mode overrides in `globals.css`. The `bg-card`, `border`, `text-muted-foreground` tokens automatically adapt. No phase-specific dark-mode work required.

## Component Specifications

### ResponsiveTable Primitive

New component at `src/components/ui/responsive-table.tsx`. Marked `"use client"`.

**Desktop layout (`hidden md:block` wrapper):**
- Renders existing `<Table>` component unchanged.
- `<TableHeader>` background: inherits `bg-background` (no explicit override).
- `<TableRow>` hover: `hover:bg-muted/50` (existing Table component handles this).
- Right-aligned cells: `text-right` class on `<TableCell>` and `<TableHead>`.

**Mobile card layout (`md:hidden` wrapper):**
- Card shell: `rounded-lg border bg-card p-4 space-y-2 cursor-pointer hover:bg-muted/50 transition-colors`
  - `rounded-lg` = `var(--radius)` = 0.5rem (8px)
  - `border` = 1px solid `var(--border)`
  - `bg-card` = `var(--card)` = white surface
  - `p-4` = 16px internal padding (all sides)
  - `space-y-2` = 8px between primary field and detail grid
- Primary field: `text-sm font-medium` — full width, top of card
- Detail grid: `<dl className="grid grid-cols-2 gap-x-3 gap-y-1 text-sm">`
  - `<dt>`: `text-muted-foreground` (label)
  - `<dd>`: default foreground; `text-right font-mono tabular-nums` for numeric/currency columns
- Card list container: `space-y-3` between cards (12px gap)
- `data-testid="data-row"` on every card `<div>` — mandatory for Cypress compatibility.

**Actions column placement (for pages with DropdownMenu):**
- Actions are rendered outside the `<dl>` grid.
- Position: trailing element in the primary row using `flex justify-between items-center` on the card header `<div>`.
- The primary field text is `flex-1 text-sm font-medium`. The DropdownMenu trigger floats right.
- Source: RESEARCH.md Pattern 3 + Open Question 1 recommendation.

**Empty state:** passed via `emptyState` prop. The `ResponsiveTable` renders it when `rows.length === 0`. Each page supplies its own empty state node.

### Dashboard KPI Grid (RESP-01)

No layout change required — `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3` is already single-column at 390px. Only change: if Creditors KPI uses `sm:grid-cols-2`, update to `md:grid-cols-2` to align with the project's `md` = mobile/desktop breakpoint. Source: RESEARCH.md Pitfall 6.

## Interaction Contract

| Interaction | Trigger | Response |
|-------------|---------|----------|
| Row click (Customers, Loans, Watchlist) | Click/tap anywhere on card | Navigate to detail page via `router.push()` |
| Row click (other pages) | No row-level navigation | Cards are non-clickable; omit cursor-pointer and hover state |
| Actions menu (Loans, Payments, Creditors) | Tap DropdownMenu trigger in card header | DropdownMenuContent opens; same behavior as desktop |
| Actions menu (Expenses, Income delete) | Delete button in card | Same confirmation/toast flow as desktop |
| Card hover (clickable) | Mouse hover | `bg-muted/50` background transition (150ms, `transition-colors`) |
| Card hover (non-clickable) | Mouse hover | No hover effect |

## Per-Page Card Field Mapping

Defines `primary` and visible detail fields for each page's card layout. Source: RESEARCH.md Per-Page Column Inventory.

| Page | Primary Field | Detail Fields (in order) | Actions? | Row Clickable? |
|------|-------------|--------------------------|----------|---------------|
| Customers | Full Name | Contact, Status | No | Yes → `/customers/{id}` |
| Loans | Customer Name | Loan ID, Principal, Rate, Status, Start Date | Yes (admin: View, Edit, Delete) | No (use actions menu) |
| Payments | Customer Name | Loan Ref, Date, Amount, Interest, Principal | Yes (admin: Edit, Delete) | No |
| Creditors | Name | Contact, Address, Created | Yes (View button/link) | No |
| Expenses | Category + Date (primary) | Amount, Notes | Yes (Delete) | No |
| Income | Category + Date (primary) | Amount, Notes | Yes (Delete) | No |
| Watchlist | Customer Name | Loan Amount, Outstanding, Days Overdue, Daily Rate, Last Payment | No | Yes → `/customers/{id}` |

**Numeric columns that require `font-mono tabular-nums text-right`:** Principal, Rate, Amount, Interest, Principal Portion, Loan Amount, Outstanding Balance, Daily Rate.
