## Typography

Source: inferred from existing components — `kpi-card.tsx` uses `text-sm` (14px) labels and `text-2xl font-semibold` (24px) values; `watchlist/page.tsx` uses `text-2xl font-semibold` for page headings; `text-sm text-muted-foreground` for subtitles. Geist Sans is the declared `--font-sans`.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Table cell text, filter labels, subtitles (`text-sm`) |
| Label | 14px | 500 (medium) | 1.5 | Table column headers (`text-sm font-medium`), card labels |

Typography rule: 2 weights only — 400 for body/muted text, 600 (semibold) for headings and values. The 500 medium is used exclusively for table column headers via Tailwind's `font-medium` class (already in the Table component).

## Color

Source: `src/app/globals.css` — all values are CSS custom properties using OKLCH. The palette is neutral (no hue) for the design system base. Semantic colors (destructive, muted) are pre-defined.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` (`oklch(1 0 0)` / white) | Page background, tab content area |
| Secondary (30%) | `--card` (`oklch(1 0 0)`) + `--muted` (`oklch(0.97 0 0)`) | Summary stat cards, table header rows, date navigation bar background |
| Accent (10%) | `--primary` (`oklch(0.205 0 0)` / near-black) | Active tab trigger, primary button (if needed), focused date picker |
| Destructive | `--destructive` (`oklch(0.577 0.245 27.325)` / red) | Not applicable in this phase — no destructive actions |

Accent reserved for:
- Active `TabsTrigger` underline/indicator (the "Daily" tab when active)
- Focused state of the Calendar date picker (selected date cell)
- Prev/next day arrow button hover state

Due-today urgency badge colors (from existing `OverdueBadge` component — source: `src/components/watchlist/overdue-badge.tsx`):
- 30–59 days: `bg-red-100 text-red-800` — due now
- These are semantic utility colors layered on top of the base palette, not part of the 60/30/10 split

Muted foreground (`--muted-foreground`, `oklch(0.556 0 0)`) is used for all secondary text: card labels, table column headers, date subtitle, empty state body text.

## Layout Contract

Source: CONTEXT.md `## Implementation Decisions — Page layout`.

```
/payments?tab=daily
└── DailyCollectionsTab (single scrollable column)
    ├── Date Navigation Bar
    │   ├── [←] prev day button (ChevronLeft icon, 44px touch target)
    │   ├── Date display button → opens Calendar Popover
    │   └── [→] next day button (ChevronRight icon, 44px touch target, disabled if date = today)
    ├── Summary Cards Row (3 cards, responsive grid: 1 col mobile / 3 col desktop)
    │   ├── KpiCard: "Total Collected" — UGX amount
    │   ├── KpiCard: "Payments" — count
    │   └── KpiCard: "Average Payment" — UGX amount (or "—" if 0 payments)
    ├── Collections Breakdown Section
    │   ├── Section heading: "Collections on {date}"
    │   └── Table: customer name, loan ref, amount, interest, principal, time
    └── Due Today Section (date-independent — always shows current overdue loans)
        ├── Section heading: "Due Today"
        └── Table: customer name, loan ref, outstanding balance, days since last payment
```

Tab integration: wrap existing PaymentsClient content in shadcn `<Tabs>`. The "Daily" tab renders `<DailyCollectionsTab />`. The tab switcher sits above both the filter bar (list tab) and the date navigation (daily tab).

## Interaction Contract

### Date Navigation
- Default date on mount: today's date from `format(new Date(), 'yyyy-MM-dd')` (client-side — never server-side)
- Prev button: `subDays(currentDate, 1)` via date-fns; always enabled
- Next button: `addDays(currentDate, 1)` via date-fns; disabled when `currentDate >= today`
- Calendar: `disabled={(date) => date > new Date()}` — future dates blocked
- Date stored in URL: `?tab=daily&date=YYYY-MM-DD`; changing date updates URL, triggers re-fetch via TanStack Query key change

### Loading States
- Summary cards: render as 3 `Card` components with a `Skeleton` block in place of the value — same height as the `text-2xl` value (`h-8 w-24 rounded`)
- Breakdown table: show 3 skeleton rows (`<TableRow>` with `<TableCell><Skeleton /></TableCell>` per column)
- Due-today table: same skeleton pattern (3 rows)
- Loading skeletons appear on initial load and on every date navigation change

### Empty States
- When selected date has zero collections: show "No collections on this date" centered in the breakdown table area (see Copywriting Contract below); summary cards still render showing UGX 0 and 0 payments
- Due-today section always renders — if `getLoansDueToday()` returns an empty array, show "All loans are up to date" message (see Copywriting Contract)

### Error States
- Use `text-sm text-destructive` inline text beneath the section heading — same pattern as watchlist/page.tsx
- No toast for data-fetch errors on this page — inline display only
