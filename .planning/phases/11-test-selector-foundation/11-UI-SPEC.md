## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing — `p-4` (mobile page padding) |
| lg | 24px | Section padding — `p-6` (desktop page padding, via `md:p-6`) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Phase 11 deliverable:** Every page-level wrapper div that currently uses `p-6` must change to `p-4 md:p-6`. This is the only spacing change in this phase.

Exceptions:
- `gap-6`, `space-y-6`, `mb-6`, `mt-6`, `py-6`, `px-6`, `pl-6`, `pr-6` — directional utilities, not page padding. Leave untouched.
- Card interior padding (`rounded-lg border border-border bg-card p-6`) — component-level concern for Phase 13. Leave untouched.
- `print:p-0` blocks in receipt pages — screen padding updates to `p-4 md:p-6` are safe; print suppression remains unchanged.

## Phase 11 — Specific Implementation Contract

This section replaces the copywriting/interaction contract for a phase that is pure code hardening.

### data-testid Attribute Contract

These are the exact attribute values to use. Executors must not invent alternatives.

| Element | Attribute | Value | File |
|---------|-----------|-------|------|
| Sidebar nav element | `data-testid` | `sidebar-nav` | `src/components/layout/sidebar.tsx` line 129 |
| Data table rows (body only) | `data-testid` | `data-row` | All `<TableRow>` elements inside `<TableBody>` across all page files |

**Constraint:** Do not add `data-testid="data-row"` to `<TableRow>` elements inside `<TableHeader>`. Header rows must remain without this attribute. Violation causes Cypress length assertions to overcounting by 1 (header row included).

**Constraint:** The mobile Sheet sidebar in `app-shell.tsx` reuses the `<Sidebar>` component. When the Sheet is open, two `data-testid="sidebar-nav"` elements exist in the DOM simultaneously. This is acceptable because Cypress tests run at desktop viewport (1280×720) where the Sheet is closed. Do not attempt to differentiate the two instances in this phase — that is a Phase 12 concern.
