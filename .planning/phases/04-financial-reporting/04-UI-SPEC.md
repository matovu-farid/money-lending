## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal spacing |
| sm | 8px | Compact table cell padding, form field gaps |
| md | 16px | Default element spacing, card padding, form input padding |
| lg | 24px | Section padding, KPI card content padding (matches existing KpiCard `pt-6`) |
| xl | 32px | Layout gaps between page sections |
| 2xl | 48px | Major section breaks (e.g., between report header and data table) |
| 3xl | 64px | Page-level top padding |

Exceptions:
- Touch targets on export buttons: minimum 44px height (INFR-03 tablet target)
- KPI card grid: `gap-4` (16px) to match Phase 3 dashboard established pattern

## Color

All values are OKLCH CSS custom properties from globals.css. Do not use hex — reference the tokens directly.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` oklch(1 0 0) | Page background, all surfaces |
| Secondary (30%) | `--card` oklch(1 0 0) / `--sidebar` oklch(0.985 0 0) | Cards, sidebar, nav, table header rows |
| Accent (10%) | `--primary` oklch(0.205 0 0) | Accent reserved-for list below |
| Destructive | `--destructive` oklch(0.577 0.245 27.325) | Delete category, delete creditor, delete expense/income entry |
| Muted text | `--muted-foreground` oklch(0.556 0 0) | KPI labels, table secondary text, empty state body |
| Border | `--border` oklch(0.922 0 0) | Table borders, card borders, form input borders |

**Accent (`--primary`) reserved for:**
- Primary CTA buttons: "Add Creditor", "Add Investment", "Record Repayment", "Add Expense", "Add Income", "Generate Report"
- Export action buttons: "Export PDF", "Export Excel"
- Active sidebar nav item
- "Save" buttons in all forms

Accent is NOT used on:
- Table rows (even highlighted/hover rows use `--muted`)
- KPI card backgrounds
- Report section headings
- Filter dropdowns

**Semantic status colors for financial data:**
- Income / credit entries: `text-green-600` (inline utility only, not a token — use sparingly for debit/credit type indicators in transaction log)
- Expense / debit entries: `text-destructive` (reuse existing destructive token)

## Interaction Contracts

### Creditor Registration Form
- Field order: Name → Contact → Address → [Investment section header] → Amount Invested → Interest Rate → Date
- Interest Rate field: number input, % suffix display, default value pre-filled from system settings
- Amount field: UGX prefix display, numeric input, no decimals enforced client-side
- Submit: single "Add Creditor" button, disabled while submitting, shows spinner

### Add Investment (on creditor profile)
- Triggered by "Add Investment" button opening a Dialog (not a full page navigation)
- Fields: Amount, Interest Rate, Date
- On success: dialog closes, investment table refreshes via router.refresh()

### Record Repayment (on creditor profile)
- Triggered by "Record Repayment" button opening a Dialog
- Fields: Amount, Date
- Shows current outstanding balance as read-only reference above form
- On success: dialog closes, KPI cards and investment table refresh

### Expense / Income Entry
- Inline sheet (Sheet component from right edge) triggered by "Add Expense" / "Add Income" button
- Fields: Date, Category (select), Amount, Notes (optional textarea)
- "Add Category" inline link beneath category select — opens a small popover to type new category name
- On success: sheet closes, list refreshes

### Transaction Log Filters
- Date range: two date-picker inputs (From / To), both optional
- Type filter: select with options [All, Income, Expense] — maps to credit/debit
- Category filter: select populated from all categories (expense + income combined)
- Filters apply on change (no explicit "Apply" button) — debounced 300ms
- "Clear filters" text button appears only when any filter is active

### Report Period Picker
- Select component with options: current month + previous 11 months (12 total)
- Default selection: most recently completed calendar month
- Changing period triggers server re-fetch (Server Component navigation, not client state)

### Export Buttons
- "Export PDF" and "Export Excel" buttons appear in report page toolbar
- On click: GET request to Route Handler (`/api/reports/{type}?format=pdf|excel&period=YYYY-MM`)
- Button shows loading spinner during download
- File downloads automatically via Content-Disposition attachment header
- On error: sonner toast with message "Export failed. Try again." — no modal

### Delete Destructive Actions
- Delete category with transactions: button disabled, tooltip "Category has transactions and cannot be deleted"
- Delete expense/income entry: confirmation dialog before action (see Copywriting Contract)
- Delete creditor: not supported in Phase 4 — no delete button rendered

## PDF Export Visual Contract

The exported PDF must match these specs (implemented in `pdf.service.ts`):

| Element | Specification |
|---------|--------------|
| Page orientation | Landscape for tables, Portrait for P&L and Balance Sheet |
| Branded header | Business name (bold, 16pt), report title (12pt), period (10pt), generated date (8pt, muted) |
| Logo | Not required in Phase 4 — placeholder space reserved |
| Header separator | 1px horizontal line below header |
| Table headers | Bold, 10pt, light gray fill (#f3f4f6 equivalent) |
| Table body rows | 9pt, alternating white / very light gray rows |
| Table borders | All cells: 0.5px #e5e7eb equivalent |
| Currency formatting | "UGX {amount}" with comma thousands separator |
| Footer | Page number: "Page {n} of {total}" — bottom center |
| Font | Helvetica (jsPDF built-in, no embedding required) |

## Excel Export Visual Contract

The exported Excel workbook must match these specs (implemented in `excel.service.ts`):

| Element | Specification |
|---------|--------------|
| Header row fill | `#1f2937` background, white bold text, 12pt |
| Data rows | 11pt, alternating white / `#f9fafb` rows |
| All borders | Thin border on all sides, color `#e5e7eb` |
| Column widths | Auto-set to longest cell content, minimum 12, maximum 40 |
| Currency columns | Number format: `"UGX"#,##0.00` |
| Date columns | Format: `DD/MM/YYYY` |
| Sheet tab name | Report type + period (e.g., "P&L 2026-02") |
| Freeze pane | Row 1 (header row) frozen |
| Auto-filter | Enabled on header row |
