## Spacing Scale

Declared values (multiples of 4). Sourced from existing page patterns (p-6, space-y-4, gap-3 used throughout).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label-to-input gap (gap-1) |
| sm | 8px | Filter control internal padding, button gap |
| md | 16px | Default element spacing, sheet form gap-4 |
| lg | 24px | Page content padding (p-6 = 24px) |
| xl | 32px | Section separation within page |
| 2xl | 48px | Empty state vertical padding (py-16 = 64px top+bottom) |
| 3xl | 64px | Empty state full vertical height baseline |

Exceptions: Filter bar uses `gap-3` (12px) between filter controls, matching TransactionLogClient exactly. Touch targets for DropdownMenuTrigger use 32px (h-8 w-8), not 44px — desktop-only admin action.

## Color

Tokens are CSS variables defined in globals.css; all values use oklch. Both light and dark modes are pre-configured by the neutral preset.

| Role | CSS Variable | Light Value | Usage |
|------|-------------|-------------|-------|
| Dominant (60%) | --background | oklch(1 0 0) — white | Page background, table background |
| Secondary (30%) | --card / --muted | oklch(1 0 0) / oklch(0.97 0 0) | Filter bar surface, sheet panel, dialog surface, empty state container |
| Accent (10%) | --primary | oklch(0.205 0 0) — near-black | Primary action buttons only ("Save changes" sheet submit button) |
| Destructive | --destructive | oklch(0.577 0.245 27.325) — red | Delete payment dialog confirm button, delete DropdownMenuItem variant |

Accent reserved for: "Save changes" submit button inside the Edit Payment Sheet only. Export CSV uses `variant="outline"` (secondary action). No other elements use the primary/accent color.

Secondary semantic: `--muted-foreground` (oklch(0.556 0 0)) used for placeholder text, empty state body copy, and recorded-by column text. This is a text-only secondary, not a surface color.

## Interaction Contract

### Filter Bar

- Customer name search: `<Input>` with placeholder "Search by customer name…" — debounced 300ms before URL push, resets page to 1. Matches TransactionLogClient pattern exactly.
- Date From / Date To: `<input type="date">` native fields — label above, width w-[160px] h-8. Debounced 300ms.
- Amount Min / Amount Max: `<Input type="text" inputMode="numeric">` with UGX prefix span — width w-[120px]. Debounced 300ms.
- "Clear filters" ghost button: renders only when any filter is active. `variant="ghost" size="sm"` with `text-muted-foreground` class. Clicking resets all filters and calls `router.push("/payments")`.
- Filter bar layout: `flex flex-wrap items-end gap-3` — matches TransactionLogClient.

### Loading State

- Initial load: show `<p className="text-muted-foreground">Loading payments…</p>` centered in p-6 container (matches loans page pattern). No skeleton — consistent with existing pages.
- Mutation pending (edit/delete): DropdownMenu items are not re-disabled; toast + TanStack Query invalidation handles refresh. Use `useTransition` for delete dialog submit button disabled state.

### Edit Payment (admin only)

- Trigger: DropdownMenuItem "Edit" opens Sheet from right side.
- Sheet title: "Edit Payment"
- Fields: Date (type="date"), Amount (UGX-prefixed text input), Notes / reason (optional Textarea).
- Submit button label: "Save changes" (primary). Disabled during pending.
- On success: toast.success("Payment updated"), sheet closes, `queryClient.invalidateQueries({ queryKey: ['payments'] })`.
- On error: toast.error("Failed to update payment").

### Delete Payment (admin only)

- Trigger: DropdownMenuItem "Delete" (variant="destructive") opens Dialog.
- Dialog title: "Delete payment?"
- Dialog description: "This payment and its cascade recalculations will be reversed. This cannot be undone."
- Required reason field: `<Textarea>` labeled "Reason for deletion" — required, min 1 character to enable confirm button.
- Confirm button: variant="destructive", label "Delete payment". Disabled if reason empty or pending.
- Cancel button: variant="outline", label "Keep payment".
- On success: toast.success("Payment deleted"), dialog closes, `queryClient.invalidateQueries({ queryKey: ['payments'] })`.
- On error: toast.error("Failed to delete payment").

### Export CSV

- Button location: top-right of page header, alongside page title. `variant="outline"` (not primary — export is secondary action; primary color reserved for sheet/dialog submit).
- Button label: "Export CSV"
- Icon: `<Download className="h-4 w-4 mr-2" />` (lucide Download).
- Behavior: client-side serialization of currently-fetched rows → Blob → anchor click → filename `payments-YYYY-MM-DD.csv`.
- No loading state — synchronous browser operation.
- Button disabled when `rows.length === 0` (nothing to export).

### Pagination

- Layout: `flex items-center justify-between pt-4` (matches TransactionLogClient).
- Left: "Showing N–M of T payments" in `text-sm text-muted-foreground`.
- Right: Previous / Next buttons, `variant="outline" size="sm"`.
- Previous disabled when page === 1. Next disabled when page * pageSize >= total.
- Pagination only renders when `total > pageSize` (25).
