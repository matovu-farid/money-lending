## Spacing Scale

Declared values (8-point scale, multiples of 4):

| Token | px Value | Tailwind Class | Usage |
|-------|----------|----------------|-------|
| xs | 4px | p-1 / gap-1 | Icon gaps, inline padding, data-density list dots |
| sm | 8px | p-2 / gap-2 | Compact element spacing, input padding |
| md | 16px | p-4 / gap-4 | Default element spacing, card content padding |
| lg | 24px | p-6 / gap-6 | Section padding, card padding |
| xl | 32px | p-8 / gap-8 | Layout gaps between major modules |
| 2xl | 48px | p-12 / gap-12 | Major section breaks, page header to content |
| 3xl | 64px | p-16 / gap-16 | Page-level gutter (intentional void) |
| 4xl | 80px | p-20 / gap-20 | Large editorial gutter — DESIGN.md §1: "large `20` (7rem) gutters to isolate critical data clusters" |

Exceptions:
- Touch targets (icon-only buttons): minimum 32px (h-8/w-8 current default) — acceptable; no change needed
- Card content padding: locked at `p-6` (24px) per DESIGN.md §5 Cards: "padding: 4 (1.4rem)" — note: DESIGN.md uses a different scale label; executor uses `p-6` (24px)
- Data-density leading dot: 4×4px square, `rounded-none` — hardcoded, not from spacing scale

## Typography

| Role | Family | Size | Weight | Line Height | Letter Spacing | Usage |
|------|--------|------|--------|-------------|----------------|-------|
| Label | Geist (sans) | 11px (0.6875rem) | 600 (semibold) | 1.4 | +0.05em (uppercase) | Table headers, card subtitles, metadata — ALL CAPS |
| Body | Geist (sans) | 14px | 400 (regular) | 1.5 | 0 | Prose text, descriptions, customer names, status labels |
| Heading | Geist (sans) | 20px | 600 (semibold) | 1.2 | -0.02em (tight) | Page section headings (h2), card titles |
| Display | Geist (sans) | 24–32px | 600 (semibold) | 1.1 | -0.02em (tight) | Page h1 titles, KPI total portfolio balance |
| Numeric | Geist Mono | (inherits context size) | 400 (regular) | (inherits) | 0 (tabular-nums) | ALL currency amounts, percentages, counts, timestamps, loan IDs |

Typography rules (non-negotiable):
- Every currency value, percentage, count, timestamp, and loan slug: `font-mono tabular-nums`
- Amount columns in tables: `text-right font-mono tabular-nums`
- Amount column headers: `text-right`
- Page headings (h1): `text-2xl font-semibold tracking-tight` (tracking-tight = -0.02em)
- Metadata labels: `text-xs font-semibold uppercase tracking-wider text-muted-foreground` — semibold weight + uppercase + +0.05em tracking differentiates labels from headings visually without a third weight
- Body text (names, descriptions): Geist sans, no mono, no uppercase

Weights declared: 400 (regular) + 600 (semibold). No other weights.

Source: DESIGN.md §3 + RESEARCH.md Typography System + Pattern 1

| Accent (10%) | Surgical strike only | #002f9c (Electric Blue) | See accent reserved list below |

### Accent Reserved For (exhaustive list — do not add to this list without design review)

1. Primary action button focus ring (`focus-visible:ring-ring/50` where `--ring` = Electric Blue)
2. Input field focus border — color shifts from `--border` (#c6c6c6) to Electric Blue on `:focus`
3. New `tertiary` button variant — text-only, no background: `text-[oklch(0.35_0.2_264)]` — used for "Execute" / "Confirm" actions exclusively
4. Data-density list leading dot — 4×4px `rounded-none` square for "Active" status indicators
5. Active sidebar link — NOT Electric Blue (use tonal shift `bg-accent text-foreground` instead)

### Anti-Patterns (these must not appear in implementation)

- `border-[color]` on section separators — use spacing + tonal difference only
- `rounded-2xl` or larger — maximum is `rounded-lg` (0.5rem). Buttons use `rounded-sm` (0.125rem)
- `shadow-*` classes — no drop shadows; use surface tier elevation instead
- Multi-color badge variants for loan status — monochromatic base + tertiary accent only
- Electric Blue on decorative/non-actionable icons — use `text-muted-foreground` instead
- `on_surface` (#1a1c1d) on `#ffffff` — use on `surface` (#f9f9fb) to reduce eye strain

Source: DESIGN.md §2, §5, §6 + RESEARCH.md Anti-Patterns

## Component Contracts

### Radius

| Component | Class | Computed Value | Rule |
|-----------|-------|---------------|------|
| Buttons | `rounded-sm` | 0.125rem | Sharp financial terminal — DESIGN.md §5 |
| Cards | `rounded-lg` | 0.5rem | Maximum allowed — DESIGN.md §6 |
| Inputs | `rounded-md` | 0.4rem (calc) | Intermediate |
| Badges | `rounded-sm` | 0.125rem | Financial terminal feel |
| Dialogs/Sheets | `rounded-lg` | 0.5rem | Card-level |

Global `--radius` must change from `0.625rem` to `0.5rem` in globals.css.

### Card Component

- Remove `ring-1 ring-foreground/10` from base class
- Tonal depth provided by placing `bg-card` (white) inside `bg-muted` (surface_container_low) wrapper
- No `shadow-*` classes
- Padding: `p-6` (24px) per DESIGN.md

### Button Component (additions required)

Add `tertiary` variant to `buttonVariants` cva:
```
tertiary: "text-[oklch(0.35_0.2_264)] bg-transparent hover:bg-accent"
```
Existing `default` variant: no class change needed — token update handles color.

### Icon-Only Button Accessibility

All icon-only buttons (buttons that render only an icon with no visible text label) must include `aria-label` matching the action label in plain language. Examples:
- Close button: `aria-label="Close"`
- Edit button: `aria-label="Edit [entity]"`
- Delete button: `aria-label="Delete [entity]"`

No icon-only button may be rendered without an `aria-label`. This applies to all instances across all ~26 files touched in this phase.

### Table Component

- Keep `border-b` on `<TableRow>` — do NOT remove (scannability requirement for loan officers)
- Reduce to: `border-b border-border/15` (15% opacity ghost border — DESIGN.md §4)
- Numeric columns: `text-right font-mono tabular-nums`
- Numeric headers: `text-right`

### Dialog / Sheet Overlay (Glassmorphism)

- Overlay: `backdrop-blur-[24px]`
- Positioner/content: `bg-white/85` (surface_container_lowest at 85% opacity)
- Source: DESIGN.md §2 "Glass & Ghost Rule"

### Sidebar

- Remove `border-r border-sidebar-border` — separation is purely tonal
- Background: `bg-sidebar` = `oklch(0.96 0 0)` (#f3f3f5, surface_container_low)
- Active link: `bg-accent text-foreground` (tonal shift only — NOT Electric Blue)
- Active link: does NOT use `--sidebar-primary` (Electric Blue reserved for focus rings only)

### Input Focus State

- Idle border: `--border` = oklch(0.8 0 0) (#c6c6c6)
- Focus border: `--ring` = oklch(0.35 0.2 264) (Electric Blue) — sharp color change, no glow
- No `ring-*` glow effect on input focus — just the border color shift

### Badges (Loan / Customer Status)

| Status | Variant | Background | Text |
|--------|---------|-----------|------|
| Active | monochromatic | `bg-secondary` | `text-foreground` |
| Paid / Completed | monochromatic dim | `bg-muted text-muted-foreground` | — |
| Overdue / Alert | destructive | `bg-destructive/10` | `text-destructive` |
- No multi-color badge variants. Monochromatic base + tertiary accent pattern only.

## Surface Tier Usage (mandatory pattern)

Three-tier surface hierarchy must be consistently applied across ALL pages:

| Tier | Token | Class | Used For |
|------|-------|-------|---------|
| Base / The Desk | surface (#f9f9fb) | `bg-background` | Page body background |
| Section Layer | surface_container_low (#f3f3f5) | `bg-muted` | Grouping modules (KPI grid wrapper, filter bars) |
| Interaction Layer | surface_container_lowest (#ffffff) | `bg-card` | Individual KPI cards, active inputs, modal backgrounds |

The "stacked cardstock" depth effect is achieved by nesting `bg-card` inside `bg-muted` inside `bg-background`. No shadows needed.

## Print Layout Protection

The `@media print` block in `globals.css` hardcodes `background: white; color: black` for receipt pages. After token changes, this block must explicitly reset tokens:

```css
@media print {
  :root {
    --background: oklch(1 0 0);   /* pure white for printing */
    --foreground: oklch(0 0 0);   /* pure black for printing */
  }
  /* existing .print-hidden and .receipt-body rules unchanged */
}
```

Receipt pages `/receipts/disbursement/[loanId]` and `/receipts/repayment/[paymentId]` must be Cypress-tested after Wave 1 token changes.

Source: RESEARCH.md Pitfall 6
