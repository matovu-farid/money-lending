# Phase 16 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Nature

Phase 16 is a **pure test-writing phase**. No new application UI is introduced. No components are added, modified, or removed. All work is confined to `cypress/e2e/*.cy.ts` files.

This UI-SPEC serves as the **test interaction contract** — it defines the visual states, element identifiers, and interaction behaviors that Cypress tests must assert against. Executors must not deviate from these contracts when writing assertions.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing, mobile page padding (p-4) |
| lg | 24px | Section padding, desktop page padding (p-6) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- **Touch targets:** All interactive elements are 44x44px minimum (WCAG 2.5.8, TOUCH-01). This is a Phase 15 implementation constraint that Cypress tests must verify.
- **Mobile page padding:** `p-4` (16px) on mobile, `md:p-6` (24px) on desktop — per Phase 11 RESP-06 implementation.
- **Bottom tab bar height:** 3.5rem (56px) fixed — `main-content-pb` utility adds this as bottom padding on mobile to prevent content overflow.
- **Safe-area bottom:** `var(--safe-bottom)` = `env(safe-area-inset-bottom, 0px)` — applied via `.safe-area-bottom` class on the bottom tab bar.

## Color

All color tokens are CSS custom properties from `globals.css`. Phase 16 tests must assert against rendered visual outcomes, not raw CSS values.

| Role | Token | Approx Value | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `--background` | oklch(0.98 0 0) ≈ #f9f9fb | Page background, surfaces |
| Secondary (30%) | `--card` / `--sidebar` | oklch(1 0 0) / oklch(0.96 0 0) | Cards, sidebar, nav, bottom tab bar background |
| Accent (10%) | `--primary` | oklch(0 0 0) = #000000 | Active tab indicator, primary buttons, active nav item |
| Destructive | `--destructive` | oklch(0.577 0.245 27.325) ≈ red-600 | Delete confirmations only |

Accent reserved for:
- Bottom tab bar active indicator (`span.bg-primary` with `opacity-100`)
- Active tab label (`text-primary` class on the selected tab button)
- Primary action buttons ("Save", "Submit", "Confirm")
- Active sidebar nav item highlight

Muted foreground for inactive tabs: `text-muted-foreground` class — `oklch(0.42 0 0) ≈ #474747`.

## Interaction States — Test Contract

This section defines the exact element identifiers and CSS states that Cypress tests must assert. These are established contracts from Phases 12–15 and must not be re-invented.

### Bottom Tab Bar

| State | Selector | Assertion |
|-------|----------|-----------|
| Tab bar visible at mobile | `[data-testid='bottom-tab-bar']` | `.should("exist").should("have.css", "display", "flex")` |
| Tab bar absent at desktop | `[data-testid='bottom-tab-bar']` | `.should("not.be.visible")` or `hidden` at md+ |
| Active tab | `[data-testid='bottom-tab-{name}']` | `.should("have.class", "text-primary")` |
| Inactive tab | `[data-testid='bottom-tab-{name}']` | `.should("have.class", "text-muted-foreground")` |
| Active indicator dot | `[data-testid='bottom-tab-{name}'] span.bg-primary` | `.should("have.class", "opacity-100")` |
| Safe-area class | `[data-testid='bottom-tab-bar']` | `.should("have.class", "safe-area-bottom")` |

Tab testid names: `bottom-tab-dashboard`, `bottom-tab-customers`, `bottom-tab-payments`, `bottom-tab-loans`, `bottom-tab-more`.

### More Sheet

| State | Selector | Assertion |
|-------|----------|-----------|
| Sheet open | `[data-testid='more-sheet']` | `.should("be.visible")` |
| Sheet closed | `[data-testid='more-sheet']` | `.should("not.exist")` |
| Secondary nav items | `[data-testid='more-item-{name}']` | `.should("be.visible")` for each |

More item testid names: `more-item-creditors`, `more-item-expenses`, `more-item-income`, `more-item-reports`, `more-item-watchlist`.

### Sidebar (desktop only)

| State | Selector | Assertion |
|-------|----------|-----------|
| Sidebar visible at desktop | `[data-testid='sidebar-nav']` | `.should("be.visible")` |
| Sidebar hidden at mobile | `[data-testid='sidebar-nav']` | `.should("not.be.visible")` |

### Responsive Table / Card Dual DOM

| State | Selector | Assertion |
|-------|----------|-----------|
| Table container at desktop | `[data-slot='table-container']` | `.should("be.visible")` |
| Table container at mobile | `[data-slot='table-container']` | `.should("not.be.visible")` |
| Data rows at mobile | `[data-testid='data-row'].filter(":visible")` | `.should("have.length.gte", 1)` |
| Data rows at desktop | `[data-testid='data-row']` | `.should("have.length.gte", 1)` |

### DrawerDialog (CRUD forms)

| Viewport | Slot | Assertion |
|----------|------|-----------|
| Mobile | `[data-slot='drawer-dialog-content']` | `.should("be.visible")` with `{ timeout: 5000 }` |
| Desktop | `[data-slot='dialog-content']` | `.should("be.visible")` with `{ timeout: 5000 }` |

### FilterPanel

| State | Selector | Assertion |
|-------|----------|-----------|
| Toggle button | `[aria-label='Toggle filters']` | `.should("be.visible")` at mobile |
| Panel hidden (mobile default) | `[data-slot='filter-panel-content']` | `.should("not.be.visible")` |
| Panel visible (after toggle) | `[data-slot='filter-panel-content']` | `.should("be.visible")` |
| Panel visible (desktop default) | `[data-slot='filter-panel-content']` | `.should("be.visible")` without toggle |
