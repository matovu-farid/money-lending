## Spacing Scale

Declared values (must be multiples of 4). This project uses Tailwind v4 utility names directly.

| Token | px value | Tailwind class | Usage |
|-------|----------|----------------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon gaps, inline element padding |
| sm | 8px | `gap-2`, `p-2` | Compact element spacing, table cell padding |
| md | 16px | `gap-4`, `p-4` | Default element spacing, card interior, form spacing |
| lg | 24px | `gap-6`, `p-6` | Section padding on desktop (md+ breakpoint) |

Exceptions:
- Responsive page padding: `p-4 md:p-6` — established in Phase 11 (RESP-06, locked decision)
- Filter panel toggle button gap: `gap-1` (4px) between icon, label, and chevron — compact affordance

- Table cell padding: `p-2` (8px) — fixed per `table.tsx` existing `TableCell` style
- FilterPanel trigger button height: `h-8` (32px) — matches default Button size, does NOT qualify as a 44px touch target (that is Phase 15 scope)

## Component Inventory for This Phase

### New Component: FilterPanel (`src/components/ui/filter-panel.tsx`)

**What it does:** Wraps filter controls in a collapsible panel. On mobile (< 768px), collapsed by default with a "Filters" toggle button. On desktop (768px+), always open with the toggle hidden via CSS.

**Interaction states:**

| State | Visual |
|-------|--------|
| Mobile — collapsed (default) | Toggle button visible: Filter icon + "Filters" label + ChevronDown. Panel hidden. |
| Mobile — expanded | Toggle button visible: Filter icon + "Filters" label + ChevronUp (rotated 180deg). Panel visible with `overflow-hidden` animation via @base-ui/react Collapsible. |
| Desktop — always open | Toggle button: `md:hidden` (absent from layout). Panel always visible regardless of `open` state via `md:!block` CSS override. |
| With active filter count | Badge showing count (e.g., "3") appended to toggle label: "Filters · 3". Badge uses `--primary` bg (black). |

**DOM structure (prescriptive):**
```
Collapsible.Root (open state managed by useState)
  Collapsible.Trigger [className="flex items-center gap-1 text-sm font-medium text-muted-foreground md:hidden h-8 px-2"]
    <Filter size={16} />
    "Filters"
    {activeCount > 0 && <Badge>{activeCount}</Badge>}
    <ChevronDown [rotates to ChevronUp when open] />
  Collapsible.Panel [data-slot="filter-panel-content"] [className="md:!block overflow-hidden"]
    {children}  ← existing filter controls unchanged
```

**data-slot value:** `filter-panel-content` — used for Tailwind v4 fallback CSS targeting if `md:!block` override fails (per RESEARCH.md Pitfall 3).

### Modified Component: ResponsiveTable desktop wrapper (RESP-05)

**Change:** Desktop wrapper div gets `max-h-[calc(100vh-12rem)] overflow-y-auto`. TableHead calls in `responsive-table.tsx` receive `className={cn("sticky top-0 bg-background z-10", ...)}`.

**Visual contract:**
- Sticky header background: `bg-background` (`--background` token, #f9f9fb) — matches page background for list pages
- Header `z-index`: 10 — above table body rows, below modals (modals portal to `document.body`)
- Scroll container max height: `calc(100vh-12rem)` — accounts for top bar, page heading, and filter panel area. If this clips content unexpectedly during execution, executor may tune to `max-h-[70vh]` as safe fallback (per RESEARCH.md Open Question 2)

### Unchanged: All form layouts (RESP-03)

No visual changes. All forms already use `space-y-4` single-column stacking. Phase deliverable is Cypress tests only.

## Interaction Contracts

### FilterPanel Toggle Animation

- Chevron rotation: `transition-transform duration-150 ease-out` — 150ms, CSS-only
- Panel open/close: @base-ui/react Collapsible.Panel built-in animation via CSS custom properties — do not hand-roll `max-height` animation
- No JS viewport detection. CSS `md:hidden` / `md:!block` controls visibility by breakpoint

### Sticky Header Scroll Behavior

- Headers pin to top of the scroll container (not the browser window)
- `position: sticky; top: 0` on `<th>` elements
- Scroll container must have `overflow-y: auto` and a bounded `max-height` — without this, sticky has no effect (see RESEARCH.md Pitfall 1)
- Dropdown menus in the Actions column use `@base-ui/react/menu` which portals to `document.body` — they will NOT be clipped by `overflow-y-auto` (confirmed safe)

## Implementation Notes for Executor

1. **Tailwind v4 `!important` syntax:** In Tailwind v4, `!block` generates `display: block !important`. Use `md:!block` on `Collapsible.Panel`. If this override does not work in the browser, fall back to a `globals.css` media query targeting `[data-slot='filter-panel-content']`.

2. **@base-ui/react Collapsible import path:** `import { Collapsible } from "@base-ui/react/collapsible"` — sub-components are `Collapsible.Root`, `Collapsible.Trigger`, `Collapsible.Panel`. Follow the same pattern as `src/components/ui/popover.tsx`.

3. **Background colour for sticky header:** Use `bg-background` (`--background` token). List pages render on `bg-background`. If a future page renders the table on `bg-card`, the executor must pass `bg-card` explicitly at that call site.

4. **No JS viewport detection:** Consistent with Phase 12 and 13 locked decisions. FilterPanel `open` state initialises to `false` (mobile collapsed). Desktop always-open is achieved via CSS only (`md:!block`), not `useEffect + matchMedia`.

5. **Cypress test for single-column forms (RESP-03):** Assert that at 390px viewport, adjacent form inputs do NOT share the same `top` coordinate (`getBoundingClientRect`), or assert the form container fills viewport width without horizontal scroll. See RESEARCH.md Pitfall 5.
