  <action>
Create `src/services/report.service.ts` with these exported functions:

**1. `getPnlData(period: string)`** — `Effect.Effect<PnlData, DatabaseError>`:
- `period` is "YYYY-MM" format
- Parse to get start/end of month dates
- Query `transactions` table filtered by `transactionDate` within the month, joined with `transactionCategories` for category name
- Group by category, sum amounts
- Separate into income (type="credit") and expense (type="debit") arrays
- Calculate totalIncome (sum of all credit amounts), totalExpenses (sum of all debit amounts), netProfit = totalIncome - totalExpenses
- All arithmetic uses BigNumber — `new BigNumber(amount)` for each row, `.plus()` for aggregation, `formatAmount()` for output

**2. `getBalanceSheetData(asOf: string)`** — `Effect.Effect<BalanceSheetData, DatabaseError>`:
- `asOf` is "YYYY-MM-DD" or "YYYY-MM" (last day of month)
- **Assets**: Total loans outstanding = SUM of all active loans' `principalBalanceAfter` from their most recent non-deleted payment (or `principalAmount` if no payments). This matches the pattern in `dashboard.service.ts` for `loansOutstanding`.
- **Liabilities**: Total creditor balances = SUM across all creditor investments of (`principalBalance` + interest accrued since last repayment). Use `getSystemCapital()` from creditor.service.ts for the `totalOutstanding` figure.
- **Equity**:
  - Share Capital = SUM of all transactions where category name = "Share Capital" and type = "credit" up to `asOf` date
  - Retained Earnings = SUM of all historical P&L net profits (total credit transactions - total debit transactions) up to `asOf` date
  - totalEquity = shareCapital + retainedEarnings
- Log a warning if Assets != Liabilities + Equity (Pitfall 3 from RESEARCH.md) — `console.warn("Balance sheet imbalance: ...")`

**3. `getPortfolioData()`** — `Effect.Effect<PortfolioEntry[], DatabaseError>`:
- Query all active loans joined with customers for customerName
- For each loan, compute:
  - outstandingBalance: `principalBalanceAfter` from most recent non-deleted payment (or `principalAmount` if no payments)
  - interestAccrued: `calculateInterest(outstandingBalance, loan.interestRateMonthly, daysElapsedSinceLastPayment)` from engine.ts
  - daysOverdue: `calculateDaysOverdue(totalInterestAccrued, totalInterestPaid, dailyRate)` from engine.ts
  - riskFlag: `daysOverdue >= 30` (matches watchlist threshold)
  - status: loan.status
- Order by daysOverdue descending (most at-risk first)

**4. `generateMonthlySnapshot(period: string, generatedBy: string)`** — `Effect.Effect<void, DatabaseError>`:
- Check if snapshot already exists for this period+type (idempotency guard per Pitfall 8)
- If not, generate P&L and Balance Sheet data
- Insert two rows into `financialSnapshots` table: one for type="pnl", one for type="balance_sheet"
- Data stored as JSON.stringify of PnlData / BalanceSheetData
- This function is called by the month-end cron endpoint (Plan 08)

Create tests in `src/services/__tests__/report.service.test.ts`:
- Test P&L aggregation math
- Test balance sheet identity (A = L + E)
- Test snapshot idempotency
- Test portfolio risk flag threshold
